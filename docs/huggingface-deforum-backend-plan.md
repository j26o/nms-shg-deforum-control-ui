# Hugging Face Deforum Backend Plan

Status: local proxy, UI adapter, and private Docker Space smoke endpoint implemented; real remote A1111 Deforum runtime pending
Last updated: 2026-05-19

Approval: Proceed with this direction. Hugging Face is an optional Deforum-compatible backend for the shared image-keyframe preset contract, not a generic video-generation path.

## Why

The prototype may need a cloud option for actual Deforum-like output when the Windows RTX 4090 workstation is unavailable, busy, or difficult to reproduce across collaborators. This option must preserve the same Deforum-style image-keyframe workflow as the local Automatic1111 backend; it is not a generic text-to-video experiment.

## What

Add Hugging Face as an optional render backend that accepts the same reviewed preset contract used by the workbench, runs a Deforum-compatible image-to-image animation pipeline remotely, and returns MP4 output plus settings/job metadata for take comparison.

The intended backend is either:

- a dedicated Hugging Face Inference Endpoint running a custom Deforum-compatible handler; or
- a private Hugging Face Space/API that wraps Automatic1111 Deforum, a Deforum-compatible render path, or an equivalent Diffusers-based image-keyframe loop.

## Source References

- Hugging Face Inference Providers support image and video generation tasks through the Hugging Face clients: `https://huggingface.co/docs/inference-providers/index`
- Hugging Face text-to-video providers exist, but the stock task is prompt-led and is not sufficient by itself for this prototype's frame-keyed Deforum workflow: `https://huggingface.co/docs/inference-providers/en/tasks/text-to-video`
- Hugging Face Inference Endpoints can expose dedicated private APIs and direct endpoint URLs through `InferenceClient`: `https://huggingface.co/docs/huggingface_hub/main/guides/inference`
- Custom diffusion/image-to-image behavior may require a custom handler on the endpoint rather than a stock task: `https://discuss.huggingface.co/t/image-to-image-stable-diffusion-inference-endpoint/24438`

## Context

Relevant local files:

- `src/services/renderAdapter.js`: normalised preset contract shared by render backends.
- `src/services/a1111DeforumAdapter.js`: current local Deforum settings translator and closest implementation model.
- `src/stores/usePresetStore.js`: render queue and take creation flow.
- `src/components/workbench/Workbench.jsx`: render actions and take metadata display.
- `src/config/defaultPreset.js`: image-reference-led timeline using bundled 1680x720 source images.
- `docs/render-adapter-contract.md`: backend boundary and metadata expectations.
- `docs/export-format.md`: reviewed preset and Deforum settings export contract.

Patterns to follow:

- Keep the UI talking to a normalised preset, not provider-specific payloads.
- Preserve the existing take metadata shape: backend id, model/checkpoint, seed, resolution, frame count, FPS, render duration, output path, settings path or payload, and logs.
- Keep image keyframes primary. Text prompts are allowed as conditioning, but they must not replace the supplied pre-rendered image references.

## Constraints

### Must

- Produce actual Deforum-like output from frame-keyed image references.
- Accept source images, timeline frame ranges, prompts/negative prompts, generation settings, motion schedules, model profile, seed, FPS, preview resolution, and output duration.
- Return or expose an MP4 artifact by default.
- Return machine-readable job metadata and enough settings to reproduce the run.
- Keep Hugging Face credentials out of browser JavaScript. Use a local or server proxy with `HF_TOKEN` in environment variables.
- Support local files by uploading image payloads or by using signed/private URLs generated by the proxy.
- Keep local Automatic1111 as the baseline backend for comparison and fallback.

### Must Not

- Do not use stock text-to-video as the main implementation path unless it can accept the image keyframe sequence and preserve Deforum-style motion/config controls.
- Do not send source assets to a public endpoint without an explicit review of privacy and client/content constraints.
- Do not store `HF_TOKEN`, endpoint URLs with secrets, or temporary signed upload URLs in committed files.
- Do not change the exported preset schema in a way that breaks local Automatic1111 rendering.

### Out Of Scope

- Public visitor-facing cloud rendering.
- Production show-control integration.
- Generic prompt-only video generation.
- Training or fine-tuning custom models.
- Replacing the local Automatic1111 backend.

## Proposed Architecture

```text
React/Vite Workbench
  -> render adapter selector: Mock | Local A1111 Deforum | Hugging Face Deforum
  -> local proxy: /hf-deforum/*
  -> Hugging Face dedicated endpoint or private Space API
  -> remote Deforum-compatible render job
  -> MP4 + settings/job metadata
  -> take comparison and export flow
```

The browser should only call the local proxy. The proxy owns authentication, upload preparation, job submission, polling, artifact download, and error normalisation.

- `src/services/huggingFaceDeforumAdapter.js`: frontend adapter that sends the simplified image-keyframe preset payload to the local proxy and normalises remote job metadata into the take flow.
- `src/stores/usePresetStore.js`: backend selection between `a1111-deforum` and `huggingface-deforum`.
- `src/components/workbench/Workbench.jsx`: toolbar backend selector and Hugging Face render action label.
- `.env.example`: local configuration keys for the proxy and endpoint paths.
- `remote/huggingface-deforum-handler/`: deployable private Docker Space/API runtime exposing `/jobs`, `/jobs/:id`, and `/jobs/:id/artifact`.
- Private Hugging Face Space `robaldovino/nms-shg-deforum-endpoint`: deployed in smoke mode at `https://robaldovino-nms-shg-deforum-endpoint.hf.space`.

Pending:

- a reachable remote Automatic1111 Deforum service for the private Space to call through `HF_DEFORUM_A1111_BASE_URL`;
- a real Deforum MP4 artifact eval against the same preset as the local A1111 baseline.

Token handling:

- The proxy reads `HF_TOKEN`, `HUGGING_FACE_HUB_TOKEN`, or a named Hugging Face CLI cache token.
- The default named token is `nms-shg`; override with `HF_TOKEN_NAME`.
- The token is never sent to browser JavaScript.

## Payload Contract

The Hugging Face backend should receive a versioned payload derived from `normaliseRenderConfig()`:

```json
{
  "schemaVersion": "0.1.0",
  "backend": "huggingface-deforum",
  "presetName": "future-wall-morph-study-01",
  "target": {
    "sourceResolution": [1680, 720],
    "previewResolution": [896, 384],
    "aspectRatio": "7:3",
    "fps": 24,
    "durationSeconds": 10
  },
  "model": {
    "modelId": "sd15-baseline",
    "file": "v1-5-pruned-emaonly.safetensors"
  },
  "assets": [
    {
      "id": "image-001",
      "uploadRef": "hf-upload-or-endpoint-local-path",
      "width": 1680,
      "height": 720
    }
  ],
  "timeline": [
    {
      "fromFrame": 0,
      "toFrame": 9,
      "sourceImageId": "image-001",
      "prompt": "Use Source 01 as the primary visual reference frame.",
      "negativePrompt": "text artifacts, flicker, broken geometry"
    }
  ],
  "generation": {
    "sampler": "DPM++ 2M Karras",
    "steps": 25,
    "cfgScale": 7,
    "seed": 123456
  },
  "motion": {
    "zoom": 1.02,
    "panX": 0,
    "panY": -0.01,
    "rotation": 0,
    "depthWarpStrength": 0.3,
    "cadence": 2
  },
  "output": {
    "format": "mp4",
    "artifactPolicy": "return-url-or-download"
  }
}
```
